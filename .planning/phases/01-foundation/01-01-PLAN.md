package com.dietcode.config

import org.springframework.context.annotation.Configuration
import org.springframework.web.servlet.config.annotation.CorsRegistry
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer

@Configuration
class CorsConfig : WebMvcConfigurer {
    override fun addCorsMappings(registry: CorsRegistry) {
        registry.addMapping("/api/**")
            .allowedOrigins("http://localhost:5173")
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .allowCredentials(false)
            .maxAge(3600)
    }
}

## Trust Boundaries

| Boundary | Description |

| Browser → RecipeController (POST /api/recipe/transform) | Untrusted JSON body crosses from browser into the Spring MVC layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `TransformRequest` deserialization | mitigate | Jackson with `jackson-module-kotlin` validates field types at deserialization; malformed JSON returns 400. Phase 2 adds `@Valid` + Bean Validation on `input` (non-blank). Phase 1 stub is safe because hardcoded response is returned regardless of body content. |
| T-01-02 | Information Disclosure | `application.yml` ANTHROPIC_API_KEY | accept | Key uses `${ANTHROPIC_API_KEY:placeholder-not-needed-in-phase-1}` default. Phase 1 makes no LLM calls. The placeholder value is never sent to Anthropic. The `.gitignore` (generated by Initializr) excludes IDE files; `application.yml` itself contains no secrets — the actual key is only in the environment variable. |
| T-01-03 | Denial of Service | CORS preflight OPTIONS flood | accept | DietCode is a personal-use single-user app with no external exposure. No reverse proxy or rate limiter in Phase 1 scope. `maxAge(3600)` in CorsConfig reduces preflight requests by caching browser-side for 1 hour. |
| T-01-04 | Spoofing | CORS origin bypass | mitigate | `allowedOrigins("http://localhost:5173")` is an exact string match — Spring rejects requests from any other origin. `allowCredentials(false)` prevents cookie/credential leakage. Wildcard `*` is NOT used. |
</threat_model>
