## Spacing Scale

Carried forward from Phase 4 without change. All values are multiples of 4, expressed as Tailwind v4 utilities.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps (`Info` icon `ml-1`), badge padding |
| sm | 8px | `gap-2` / `p-2` | Inline element spacing, toolbar button gap, tag chip padding |
| md | 16px | `gap-4` / `p-4` | Default section spacing, card padding (`CardContent`) |
| lg | 24px | `gap-6` / `p-6` | Column gap in comparison layout, section breaks |
| xl | 32px | `gap-8` / `p-8` | Layout vertical rhythm, `mt-8` between form and result |
| 2xl | 48px | `py-12` | Major section separators |
| 3xl | 64px | `py-16` | Page-level top/bottom padding (future; not currently used) |

Exceptions:
- Touch targets (stepper `+`/`-` buttons): `min-w-9 h-9` = 36px minimum. No change.
- Stepper display cell: `min-w-12` = 48px. No change.
- Textarea: 10 rows default (`rows={10}`). Not spacing-token controlled.

### Polish additions for Phase 5

The existing `<main className="px-4 py-8">` gives 16px horizontal / 32px vertical page padding. This is correct for mobile but produces a sparse page on desktop. Phase 5 refines this with responsive padding:

- Mobile (default): `px-4 py-8` ‚ÄĒ unchanged
- Tablet and above (`sm:`): `sm:px-6` ‚ÄĒ 24px horizontal
- Desktop (`lg:`): `lg:px-8` ‚ÄĒ 32px horizontal

The `h1` title currently has `mb-8` (32px below). Retain at `mb-8` ‚ÄĒ correct visual weight.

## Typography

Inherited from Phase 4. Two weights only: 400 (normal) and 700 (bold). No changes to the type scale.

| Role | Size | Weight | Line Height | Tailwind Class | Where Used |
|------|------|--------|-------------|----------------|------------|
| Body | 14px | 400 | 1.5 | `text-sm` | Ingredient list items, instruction steps, labels, error text, popover content |
| Label | 14px | 400 | 1.4 | `text-sm text-muted-foreground` | Form field labels (`"Your Recipe"`, `"Diet profiles"`), stepper original count, URL badge |
| Heading (section) | 16px | 700 | 1.4 | `text-base font-bold` or `font-semibold` | `"Ingredients"`, `"Instructions"` within RecipeCard; column headers `"Original"` / `"Adapted"` |
| Display (recipe name) | 24px | 700 | 1.2 | `text-2xl font-bold` | `<h1>DietCode</h1>` page title; `CardTitle` recipe name |

Notes on current inconsistencies to fix in Phase 5:
- `RecipeCard.tsx` uses `font-semibold` for "Ingredients" and "Instructions" headings (`h3`). Phase 4 spec declared `font-bold` for section headings. Phase 5 must align these to `font-bold` for consistency.
- The page `<h1>` is `text-2xl font-bold` ‚ÄĒ correct, keep as-is.
- `CardTitle` (from shadcn `card.tsx`) uses its own typography styles. Do not override shadcn internals; let `CardTitle` inherit its default weight.

Print typography (existing, unchanged):
- Body: `12pt`, `line-height: 1.5`
- Headings: `14pt`, `font-weight: 700`
- Recipe name: `18pt`, `font-weight: 700`
- Page margins: `1cm` all sides via `@page`

## Color

Uses shadcn CSS variables exclusively. No raw hex values except the existing amber warning block (preserved from Phase 2).

| Role | CSS Variable | Approximate Zinc Value | Usage |
|------|-------------|------------------------|-------|
| Dominant (60%) | `bg-background` | White / near-white | Page background, card interior, textarea, tag input, stepper display, button backgrounds (secondary style) |
| Secondary (30%) | `bg-muted` / `border-border` | Zinc-100 / Zinc-200 | Card borders, hover states on muted buttons, tag chip background, stepper button hover, URL badge, column dividers (border only) |
| Accent (10%) | `bg-primary` / `text-primary-foreground` | Zinc-900 | Submit button, active diet pills, Print Recipe button, `Loader2` spinner |
| Destructive | `text-destructive` | Red-500-range | Error paragraph (`‚ö† {error}`) in form, Drive export error paragraph |

Accent (`bg-primary`) reserved for exactly these elements ‚ÄĒ nothing else:
1. "Transform Recipe" submit button (full `bg-primary`)
2. Active/selected diet pills (`bg-primary text-primary-foreground border-primary`)
3. "Print Recipe" button in ExportToolbar (`bg-primary text-primary-foreground`)
4. `Loader2` spinner inside "Transform Recipe" button during loading

`bg-muted` / secondary style applies to:
- "Export to Drive" button
- Unselected diet pill hover state
- ServingStepper `+`/`-` button hover
- Tag chip background

Preserved exception: warning block ‚ÄĒ `bg-amber-50 border border-amber-200` ‚ÄĒ not changed in Phase 5.

Diet label badges on the adapted column:
- Selected diets: `bg-primary/10 text-primary` (semi-transparent accent, correct)
- Intolerances: `bg-muted text-muted-foreground border` (secondary, correct)

## Layout Contract

### Responsive Breakpoints

| Breakpoint | Screen Width | Changes |
|------------|-------------|---------|
| Default (mobile) | < 640px (`sm`) | Single column everywhere; form full-width; comparison stacked; `px-4` page padding |
| `sm` | ‚Č• 640px | `px-6` page padding |
| `md` | ‚Č• 768px | Comparison layout switches to two-column (`flex-row`); diet pills wrap naturally |
| `lg` | ‚Č• 1024px | `px-8` page padding; `max-w-5xl` comparison container comfortably centered |

### Form layout (mobile-first refinements)

Current form container: `max-w-2xl mx-auto` ‚ÄĒ correct, no change.

- On mobile, form sections have `space-y-6` vertical rhythm ‚ÄĒ correct, no change.
- Textarea `rows={10}` is appropriate for paste input on all screen sizes ‚ÄĒ no change.
- Submit button: `w-full` ‚ÄĒ correct for mobile; acceptable on desktop (matches form width).

### Comparison layout

- Container: `max-w-5xl mx-auto flex flex-col md:flex-row gap-6` ‚ÄĒ correct, no change.
- Each column: `flex-1 min-w-0` ‚ÄĒ correct, prevents text overflow on narrow columns.
- On mobile (< `md`): Original column renders first, Adapted column below ‚ÄĒ correct reading order.
- Column gap: `gap-6` (24px) ‚ÄĒ no change.

### App.tsx main element

Current: `<main className="px-4 py-8">`

Phase 5 target: `<main className="px-4 sm:px-6 lg:px-8 py-8">`

This is the only structural layout change in Phase 5 for the main wrapper.

### ExportToolbar placement

Between "Edit / start over" and `<ComparisonLayout>`. Already using `max-w-5xl mx-auto mb-6 print:hidden`. No change needed.

## State Inventory

| State | Component | Visual Treatment |
|-------|-----------|-----------------|

| Error (scraping fail) | Below form | Same style as transform error |

| Original unavailable (URL import, no data) | Original column | `text-sm text-muted-foreground` paragraph |
| Drive connecting | ExportToolbar | Secondary button + `Loader2` + "Connecting..." label + `aria-busy` |
| Drive exporting | ExportToolbar | Same as connecting + "Exporting..." label |
| Drive error | ExportToolbar | `role="alert"` paragraph below Drive button, `text-destructive` |
| Diet pill ‚ÄĒ active | DietPillGroup | `bg-primary text-primary-foreground border-primary` |
| Diet pill ‚ÄĒ inactive | DietPillGroup | `bg-background text-foreground border-border hover:bg-muted` |

### Loading state visual contract

The loading state (LLM in-flight) is the most important feedback moment. Current implementation:
- Button shows `Loader2 animate-spin` + "Transforming..."
- Form remains visible (not collapsed) until response arrives
- No skeleton, no progress indicator beyond the button state

Phase 5 adds no new loading animation beyond what already exists. The button state is sufficient for a single-request flow.

## Responsive Design Contract

- Page: `px-8 py-8`

## Phase 5 Specific Refinements Summary

The following are the concrete, actionable changes Phase 5 must make. Everything else is a hold-steady:

1. **Responsive page padding** ‚ÄĒ `<main>`: `px-4 py-8` ‚Üí `px-4 sm:px-6 lg:px-8 py-8`
2. **Typography consistency** ‚ÄĒ `RecipeCard.tsx` `h3` tags: `font-semibold` ‚Üí `font-bold` to match section heading spec
3. **Instruction list spacing** ‚ÄĒ `RecipeCard.tsx` instruction `ol`: add `leading-relaxed` to list items for improved readability
4. **Error accessibility** ‚ÄĒ `App.tsx` error `<p>`: add `role="alert"` for screen reader announcement
5. **Empty state copy** ‚ÄĒ `App.tsx`: "Original recipe unavailable." ‚Üí "Original recipe unavailable ‚ÄĒ paste it above to see the comparison."
6. **Comparison column mobile spacing** ‚ÄĒ ensure `ComparisonLayout` children have `px-4` padding protection at mobile viewport; currently `flex-1 min-w-0` may allow cards to touch viewport at narrow widths. Wrap with `min-w-0` and rely on `Card` internal `p-6` (shadcn default) for visual breathing room.
7. **Focus ring consistency** ‚ÄĒ textarea currently has `focus:ring-1 focus:ring-ring`; verify all interactive elements (pills, tag input, stepper buttons) use the same `focus-visible:ring-1 focus-visible:ring-ring` pattern for keyboard navigation consistency.
