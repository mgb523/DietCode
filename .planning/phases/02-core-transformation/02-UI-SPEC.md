## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag-to-tag gap inside TagInput, pill gap |
| sm | 8px | Compact element spacing — gap between pill row and label, gap between tags |
| md | 16px | Default element spacing — between form fields, inner card padding |
| lg | 24px | Section padding — between constraint panel sections (diet pills block, tag input block) |
| xl | 32px | Layout gaps — between form card and result card |
| 2xl | 48px | Major section breaks — not used in Phase 2 |
| 3xl | 64px | Page-level spacing — not used in Phase 2 |

Exceptions:
- Touch target for pill toggle buttons: minimum 36px height (px-3 py-1 at 14px font satisfies this; verify tappability on mobile)
- Tag remove button: 20px hit area minimum — use padding to expand beyond the × glyph

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Heading (section) | 16px | 700 (bold) | 1.25 |
| Display (page title, recipe name) | 24px | 700 (bold) | 1.2 |

Notes:
- "Body" covers ingredient lines, instruction steps, placeholder text, inline error copy, form field labels, and RecipeCard section subheadings. All non-heading text uses weight 400.
- "Heading" covers the page `<h1>DietCode</h1>` at 24px and RecipeCard section subheadings ("Ingredients", "Instructions", "Notes") at 16px — both use weight 700.
- "Display" covers `recipe.recipeName` inside RecipeCard (24px, weight 700).
- Label roles ("Your Recipe", "Diet profiles", "Ingredients to omit or replace") use 14px weight 400 — distinction from body text is communicated by context and position, not weight.
- Ingredient quantity+unit span: same 14px body size, weight 700 (bold) per D-08. This is not a separate type role — it is a bold variant within the body line.
- Source: shadcn New York defaults mapped to Tailwind utility classes: `text-sm` (14px), `text-base` (16px), `text-2xl` (24px); weights `font-normal` (400), `font-bold` (700).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` ≈ zinc-50 in light / zinc-950 in dark | Page background, form background |
| Secondary (30%) | `hsl(var(--card))` ≈ white in light / zinc-900 in dark | RecipeCard surface, form section groupings |
| Accent (10%) | `hsl(var(--primary))` ≈ zinc-900 in light / zinc-50 in dark | Selected diet pill background + text, Submit button background |
| Destructive | `hsl(var(--destructive))` ≈ red-500 | Inline error message text only |

Accent reserved for:
1. Selected state of diet toggle pills (`bg-primary text-primary-foreground border-primary`)
2. Submit / "Transform Recipe" button background
3. Focus ring on the recipe textarea and tag input (`focus-within:ring-ring`)

Color contract uses only shadcn CSS variables — never hardcoded hex values. This ensures automatic dark mode compatibility without additional work.

Warning/notes callout (recipe warnings[]): `bg-amber-50 border-amber-200` (already present in RecipeCard.tsx — retain as-is). This is a one-off semantic color for the warnings block, not part of the 60/30/10 accent budget.

## Layout Contract

### Page structure (D-01: single-page vertical, top-to-bottom)

```
<main> [max-w-2xl mx-auto px-4 py-8]
  <h1> DietCode                          [text-2xl font-bold text-center mb-8]
  <form> [space-y-6]                     — visible when !formCollapsed
    <section> Your Recipe
      <label> Your Recipe                [text-sm font-normal]
      <textarea>                         [10 rows, resize-y, w-full]
    <section> Dietary Needs              [space-y-4]
      <label> Diet profiles              [text-sm font-normal]
      <DietPillGroup>                    [flex flex-wrap gap-2]
      <label> Ingredients to omit…      [text-sm font-normal mt-4]
      <TagInput>
    <button type="submit">               [w-full]
      default: "Transform Recipe"
      loading: <Loader2 animate-spin /> + "Transforming..."
    <p> error message                    [text-sm text-destructive mt-2] — visible when error
  <div> [mt-8]                           — visible when formCollapsed && recipe
    <button> "Edit / start over"         [text-sm text-muted-foreground underline mb-4]
    <RecipeCard>
```

### Form collapse behavior (D-02)

- On successful transformation: `setFormCollapsed(true)`. Form section hidden (`hidden` class or conditional render). Result area shown.
- "Edit / start over" button: sets `setFormCollapsed(false)`, clears `recipe`, clears `error`. Does NOT reset `recipeText`, `selectedDiets`, or `intolerances` — user edits from where they left off.
- Page does not scroll to top on collapse — the result card naturally appears below where the form was.

### Focal point

Primary visual focal point: the recipe textarea is the first interactive element and the primary eye-anchor on load; it occupies most of the vertical form space and receives focus-ring accent color on interaction. The "Transform Recipe" button is the secondary focal point — full-width, accent-colored background, positioned at the bottom of the form to anchor the submit action.

## Interaction Contract

### DietPillGroup (D-03)

- Pills: `[Keto] [Vegan] [GF] [Paleo] [Whole30]`
- Unselected state: `bg-background text-foreground border-border hover:bg-muted`
- Selected state: `bg-primary text-primary-foreground border-primary`
- Toggle: click selects/deselects. Multiple selections allowed simultaneously.
- All pills deselected is valid — recipe transforms with no diet constraint.
- `type="button"` on each pill to prevent form submission on click.
- Values sent to backend: `"KETO"`, `"VEGAN"`, `"GLUTEN_FREE"`, `"PALEO"`, `"WHOLE30"` (enum key names, not display labels).
- Pill className: `flex items-center px-3 py-1 rounded-full text-sm border cursor-pointer` (14px, weight 400 via `text-sm font-normal`)

### TagInput (D-04)

- Label: "Ingredients to omit or replace"
- Placeholder (shown when no tags present): `peanuts, dairy, shellfish — press Enter to add`
- Tag added on: Enter key, comma key, or onBlur (when input has content).
- Tag removed on: click × button on tag, or Backspace when input field is empty (removes last tag).
- Duplicate tags silently ignored (no error).
- Tags displayed as rounded pills inside the input container, flush with the text cursor.
- Container: `flex flex-wrap gap-2 p-2 border rounded-md bg-background focus-within:ring-1 focus-within:ring-ring`
- Tag chip: `flex items-center gap-1 px-2 py-1 bg-muted rounded-full text-sm`
- Remove button: `text-muted-foreground hover:text-foreground`, aria-label `Remove {tag}`, renders `×` glyph.

### Submit button (D-06)

- Default label: "Transform Recipe"
- Loading label: `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` + "Transforming..."
- Disabled during loading (`disabled={loading}`). Prevents double-submit.
- Full width: `w-full`.
- Use shadcn `<Button>` component if available; otherwise a plain `<button>` with `bg-primary text-primary-foreground` classes.

### Textarea

- Rows: 10 (default height). User can resize vertically (`resize-y`). No horizontal resize.
- No character count displayed (discretion decision: omit — adds visual noise for a personal tool).
- Placeholder: `Paste your recipe here...`
- Validation: non-empty required before submit (disabled submit or inline "Please enter a recipe" message below textarea if submitted empty). Use HTML5 `required` attribute as first gate.

### Error state (D-07)

- Shown inline, directly below the submit button.
- Copy: `Transformation failed — please try again.`
- Prefix glyph: `⚠` (Unicode warning sign, not a Lucide icon — avoids layout shift).
- Full copy rendered: `⚠ Transformation failed — please try again.`
- Style: `text-sm text-destructive mt-2`
- Form stays fully visible for immediate retry (no collapse on error).
- Error clears when user begins a new submission (`setError(null)` at top of `handleSubmit`).

### RecipeCard ingredient lines (D-08)

- Quantity + unit: `<span className="font-bold">{ing.quantity} {ing.unit}</span>`
- Ingredient + preparation: plain body weight (`font-normal`), space-separated after the bold span.
- Edge case — missing unit (e.g. "2 eggs"): bold span renders `"2 "` with trailing space; visually correct, no special handling.
- Edge case — missing quantity (e.g. "Salt to taste"): bold span renders empty; acceptable.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title | DietCode |
| Primary CTA (default) | Transform Recipe |
| Primary CTA (loading) | Transforming... |
| Recipe input label | Your Recipe |
| Recipe input placeholder | Paste your recipe here... |
| Diet section label | Diet profiles |
| Exclusions label | Ingredients to omit or replace |
| Exclusions placeholder | peanuts, dairy, shellfish — press Enter to add |
| "Edit" link (post-success) | Edit / start over |
| Error state | ⚠ Transformation failed — please try again. |
| Empty state heading | (no empty state needed — form is always present before first submit) |
| RecipeCard: ingredients subheading | Ingredients |
| RecipeCard: instructions subheading | Instructions |
| RecipeCard: warnings subheading | Notes |
| RecipeCard: servings line | Serves {n} |

Destructive actions in Phase 2: none. "Edit / start over" discards the current result but does not destroy persistent data (the app is stateless). No confirmation dialog required.

Section heading wording rationale:
- "Your Recipe" (not "Recipe Text" or "Input") — conversational tone matching the tool's personal-use context.
- "Diet profiles" (not "Dietary restrictions" or "Diet type") — neutral, matches D-03 pill labels.
- "Ingredients to omit or replace" — exact per D-04 locked decision.
