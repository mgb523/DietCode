## Spacing Scale

Declared values (multiples of 4 only). Uses Tailwind v4 utility classes directly — no config file.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), info icon margin |
| sm | 8px | Inline element spacing (`gap-2`), toolbar button gap |
| md | 16px | Default element spacing (`gap-4`), card padding |
| lg | 24px | Section padding (`gap-6`), column gap in side-by-side layout |
| xl | 32px | Layout gaps (`gap-8`), vertical spacing between toolbar and cards |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level vertical padding (`py-16`) |

Exceptions:
- Touch targets for stepper buttons: minimum 36px × 36px (matches existing `min-w-9 h-9` = 36px in `ServingStepper.tsx`)
- Export toolbar buttons: minimum height 36px to match stepper button height for visual alignment

## Color

Uses shadcn CSS variables. No raw hex values — all colors reference the zinc-based design token system established in Phase 1.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `bg-background` | Page background, card interior |
| Secondary (30%) | `bg-muted` / `border-border` | Card borders, hover states, column dividers |
| Accent (10%) | `bg-primary` / `text-primary-foreground` | Primary CTA button ("Transform Recipe"), active diet pills, Print button |
| Destructive | `text-destructive` | Inline Drive export error message only |

Accent reserved for:
1. "Transform Recipe" submit button
2. Active/selected diet pills (already established in Phase 2)
3. Print button (primary export action)
4. Loading spinner (`animate-spin` on `Loader2`)

Google Drive export button: secondary style (`bg-background border-border hover:bg-muted text-foreground`) — not primary, because print is the more reliable fallback and should be visually dominant.

Warning block (existing): `bg-amber-50 border-amber-200` — preserved from Phase 2, not changed.

## Layout Contract

### Comparison View Width

- Form: `max-w-2xl mx-auto` — unchanged
- Comparison container: `max-w-5xl mx-auto` — expands at result view only
- Each column: `flex-1 min-w-0` — equal width within the container

### Breakpoint Behavior

| Breakpoint | Layout |
|------------|--------|
| Below `md` (< 768px) | Single column, original card first, adapted card below |
| `md` and above (≥ 768px) | Two columns side by side, `gap-6` (24px) between columns |

Tailwind pattern: `flex flex-col md:flex-row gap-6`

### Column Labels

- Left column header: `"Original"` — `text-sm font-semibold text-muted-foreground uppercase tracking-wide`
- Right column header: `"Adapted"` — same style as "Original"

### Export Toolbar Placement

Positioned between the "Edit / start over" link and the comparison cards. Full width of the `max-w-5xl` container.

Toolbar content (left to right): `[Print]` `[Export to Drive]`

On mobile (below `md`): toolbar buttons stack or wrap via `flex flex-wrap gap-2`.

## Substitution Popover Contract

Trigger: `Info` icon (lucide-react, `h-4 w-4`) placed inline after the ingredient name.

Visibility rule: the `Info` icon is rendered ONLY when `ing.substitutionNote !== null`. It is never shown for non-substituted ingredients.

Popover content:
- Line 1: `"Substituted for: {original ingredient name}"` — rendered if original ingredient data is available; omit if original data is null
- Line 2: `{substitutionNote}` — the LLM-provided rationale
- Max width: `max-w-[280px]`
- Font size: `text-sm` (14px)

Interaction: click to open, click outside or press Escape to close. Uses `Popover` (click) not `Tooltip` (hover). This satisfies the ROADMAP success criterion: "clicking it opens a popover."

Trigger icon style: `text-muted-foreground hover:text-foreground cursor-pointer inline-flex items-center ml-1`

## Export Toolbar Contract

### Print Button

- Label: `"Print"`
- Icon: `Printer` (lucide-react) at `h-4 w-4 mr-2`
- Style: primary — `bg-primary text-primary-foreground hover:bg-primary/90 rounded-md px-4 py-2 text-sm font-medium`
- Behavior: calls `window.print()`
- Print state: no loading state needed (browser dialog is synchronous from user perspective)

### Google Drive Export Button

- Label: `"Export to Drive"`
- Icon: use text label only (no Google-branded SVG needed; keep it simple)
- Style: secondary — `border border-border bg-background hover:bg-muted text-foreground rounded-md px-4 py-2 text-sm font-medium`
- States:
  - Default: `"Export to Drive"`
  - Connecting (OAuth popup open): `"Connecting..."` with `Loader2` spinner at `h-4 w-4 animate-spin mr-2`
  - Exporting (after auth, during upload): `"Exporting..."` with `Loader2` spinner
  - Success: button returns to `"Export to Drive"` (no persistent success state; Drive itself confirms)
  - Error: button returns to default state; inline error shown below the button

### Drive Export Error

Rendered as a `<p>` below the Drive button when error state is set:

```
text-sm text-destructive mt-1
```

Copy (exact): `"Drive export failed — use Print to save as PDF instead."`

No warning icon prefix — the `text-destructive` color is sufficient signal.

## Print CSS Contract

Add `@media print` rules to `src/index.css` (alongside the existing `@import "tailwindcss"`).

Elements to hide when printing:
- The form (`form` element and its wrapper)
- "Edit / start over" button
- Export toolbar (`ExportToolbar` component)
- Original recipe column (entire left column of comparison view)
- Column label for the adapted column (`"Adapted"` header)
- Browser default headers/footers: `@page { margin: 1cm; }`

Elements to preserve when printing:
- Adapted recipe card: recipe name, serving count, ingredients list, instructions, warnings block
- `ServingStepper` is hidden in print (shows scaled serving count as static text instead)

Typography in print:
- Body: `font-size: 12pt; line-height: 1.5`
- Headings: `font-size: 14pt; font-weight: 600`
- Recipe name (display): `font-size: 18pt; font-weight: 700`
- Page margins: `1cm` all sides via `@page`

Print class strategy: apply `print:hidden` Tailwind utility class to elements that should be hidden. Apply `print:block` to the adapted-only print wrapper if needed.

## Accessibility Contract

- Substitution popover trigger: `aria-label="Why {ingredient} was substituted"` on the `Info` icon button
- Print button: `aria-label="Print recipe"` if no text label visible on mobile
- Drive button: `aria-label="Export recipe to Google Drive"`
- Drive connecting/exporting states: `aria-busy="true"` on the button during async operations
- Drive error: `role="alert"` on the error `<p>` so screen readers announce it immediately
- Comparison columns: wrap each in a `<section>` with `aria-label="Original recipe"` and `aria-label="Adapted recipe"`
- Keyboard: `Popover` from shadcn inherits Radix focus trap and Escape-to-close behavior
