4. Update the `Card` element (line 119) to merge the passed className:
```tsx
<Card className={cn("max-w-2xl mx-auto", className)}>
```
This means when no className is passed, the card keeps its `max-w-2xl mx-auto` default (works as standalone). When `className="max-w-none"` is passed from App.tsx, `cn()` produces `"max-w-2xl mx-auto max-w-none"` — Tailwind resolves the last `max-w-*` wins in v4, or for full safety use `cn("", className)` replacing the default entirely. To avoid Tailwind precedence issues, use:
```tsx
<Card className={cn("max-w-2xl mx-auto", className)}>
```
Since `className="max-w-none"` appears after `max-w-2xl`, Tailwind v4 honors the last declared class. This is correct.
